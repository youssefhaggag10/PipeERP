# PipeERP

برنامج ERP محلي لإدارة مصانع المواسير البلاستيك، يعمل Offline على جهاز ويندوز أو شبكة داخلية صغيرة.

## الهدف

بناء منتج قابل للبيع لأكثر من مصنع، وليس برنامجًا خاصًا بمصنع واحد فقط.

## التقنية

- Python 3.11+
- PySide6
- SQLite في الإصدار المحلي
- واجهة عربية RTL
- قابل للتطوير لاحقًا إلى PostgreSQL أو نسخة شبكية

## الوظائف المنفذة حاليًا

- تشغيل تطبيق Desktop عربي
- تسجيل الدخول وتشفير كلمات المرور
- إدارة الأصناف والخامات والمنتجات والهالك وقطع الغيار
- إدارة الموردين والعملاء
- مخزن المصنع وتسويات المخزون
- أوامر الشراء والاستلام مع أرقام الدفعات
- أوامر البيع والتسليم مع منع الرصيد السالب
- صرف المخزون بنظام FIFO وربط البيع بالدفعات وتكلفتها الفعلية
- أرصدة وقيم الدفعات المتبقية
- تسويات مخزون بتكلفة ورقم دفعة اختياريين
- أوامر شراء وبيع متعددة البنود مع عرض تفاصيل وإجماليات المستند
- فواتير مبيعات ومشتريات تنشأ بعد التسليم أو الاستلام
- أرصدة عملاء وموردين تفصل الفواتير المعتمدة عن دفعات الأوامر المقدمة
- تحصيلات ومدفوعات جزئية مع المدفوع والمتبقي

- كارت الصنف وربط الحركات بالمورد أو العميل
- مؤشرات أولية للمخزون

- نظام Database Migrations بإصدارات متتابعة

## التشغيل

```bash
python -m venv .venv
source .venv/bin/activate  # Linux / macOS
# أو على Windows:
# .venv\Scripts\activate

pip install -r requirements.txt
```

### أول تشغيل فقط

عند إنشاء قاعدة بيانات جديدة، حدد كلمة مرور مدير النظام في متغير البيئة `PIPEERP_ADMIN_PASSWORD` قبل التشغيل.

Linux / macOS:

```bash
export PIPEERP_ADMIN_PASSWORD="choose-a-strong-password"
python main.py
```

Windows PowerShell:

```powershell
$env:PIPEERP_ADMIN_PASSWORD="choose-a-strong-password"
python main.py
```

يتم إنشاء المستخدم `admin` فقط عندما تكون قاعدة البيانات بلا مستخدمين ويكون المتغير موجودًا. في التشغيلات التالية لا يلزم إبقاء المتغير مضبوطًا.

## إعداد طابعة الفواتير

1. ثبّت تعريف طابعة `Star TSP100` على نظام التشغيل ووصلها عبر USB.
2. افتح `الإعدادات` داخل PipeERP واختر اسم الطابعة المثبتة.
3. راجع اسم المصنع والعنوان والهواتف والتذييل واللوجو وQR الخاص بالدفع.
4. سلّم أمر البيع لإنشاء فاتورة معتمدة، ثم استخدم `معاينة وطباعة الفاتورة` من شاشة
   المبيعات أو من `الحسابات ← فواتير المبيعات`.

قالب الطباعة مضبوط على رول 80mm ومساحة طباعة فعلية 72mm. لا تُنشأ فاتورة من أمر
مسودة، وأي مبلغ مسجل قبل التسليم يظهر محاسبيًا كدفعة مقدمة حتى اعتماد الفاتورة.

راجع `SPRINT1_TEST_CASES.md` لاختبارات القبول اليدوية لدورة البيانات الأساسية والمخزون والشراء والبيع.
وراجع `SPRINT2_TEST_CASES.md` لاختبارات FIFO والدفعات وتكلفة الصرف.
وراجع `SPRINT3_TEST_CASES.md` لاختبارات أوامر الشراء والبيع متعددة البنود.
