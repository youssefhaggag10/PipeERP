# PipeERP — الحزمة التنفيذية الكاملة

تحتوي هذه الحزمة على التعديلات المطلوبة في المظهر وCRM والتصنيع والمبيعات:

## المظهر

- ثيم داكن وفاتح وحسب إعداد ويندوز.
- حجم خط قابل للتعديل من 11 إلى 20.
- تكبير عناصر الواجهة من 90% إلى 140%.
- ألوان ثابتة وواضحة للتابات والجداول في Windows Light Mode.

## CRM

- مركز أنشطة أعلى البرنامج شبيه بـ Odoo.
- عداد للأنشطة المتأخرة وأنشطة اليوم.
- عرض الأنشطة القديمة غير المنفذة والجديدة والقادمة.
- فتح تبويب أنشطة CRM مباشرة من الإشعار.

## الخزينة والبنوك

- تكبير حقول كود الحساب واسم البنك/الخزينة.
- تكبير جداول الحسابات وكشف الحركة.
- تخطيط مرن مع الشاشات المختلفة.

## خلطات التشغيل في التصنيع

- أمر التصنيع يحتوي عدة خلطات تشغيل فعلية.
- صرف الخامات لكل خلطة عند تجهيزها بدل صرف الأمر بالكامل مقدمًا.
- تسجيل العدد والوزن الفعلي والكسر والتكلفة لكل خلطة.
- إغلاق الخلطة الأولى ثم إنشاء خلطة جديدة من السابقة بدون الخامة المسببة للمشكلة.
- الاحتفاظ بتاريخ وتكلفة كل خلطة بدون تعديل الإنتاج السابق.
- مخزون المنتجات التامة يحتفظ بطبقات العدد والوزن الفعلي.

## كروت الوزن والمبيعات بالكيلو

- كارتة واحدة يمكن أن تحتوي مقاسًا واحدًا أو عدة مقاسات.
- تسجيل عدد المواسير لكل مقاس والوزن الصافي الفعلي وسعر الكيلو.
- قيمة الكارتة = الوزن الفعلي × سعر الكيلو.
- توزيع داخلي للوزن المختلط بنسبة العدد × الوزن القياسي.
- المخزون ينخفض بالعدد، وسجل الوزن ينخفض بالوزن الفعلي.
- منع تسليم أمر بيع بالوزن قبل إدخال كل المواسير في كروت وزن.
- دعم المخزون القديم الذي سبق نظام طبقات الوزن دون تعطيل التسليم.

## طريقة التطبيق

انسخ مجلد `app` فوق مجلد `app` في المشروع على الفرع:

`feature/theme-crm-manufacturing-weight`

لا توجد مكتبات خارجية جديدة.

بعد النسخ شغّل:

```bash
python -m compileall app
pytest
python main.py
```

راجع `MANUFACTURING_WEIGHT_TEST_CASES.md` لاختبارات القبول اليدوية.
